# Key Vault Shared Module

Organization-standard Bicep wrapper around the
[Azure Verified Module for Key Vault](https://github.com/Azure/bicep-registry-modules/tree/main/avm/res/key-vault/vault)
(`br/public:avm/res/key-vault/vault`).

Pinned AVM version: see [`avm.version`](./avm.version).

## Layout

```
keyvault-shared-module/
├── modules/keyvault.bicep              ← AVM wrapper
├── parameters/keyvault.example.bicepparam ← example input
├── tests/deploy.validation.sh          ← sandbox validation
├── scripts/
│   ├── check-avm-version.sh            ← detects new AVM releases
│   └── diff-avm-versions.sh            ← produces param/output diff
├── .github/
│   ├── workflows/avm-update-automation.yml
│   └── copilot-instructions.md         ← guardrails for the coding agent
├── avm.version                         ← single source of truth
└── README.md
```

## Usage

```bicep
module kv 'br:<your-acr>/keyvault-shared:<tag>' = {
  name: 'kv-platform'
  params: {
    name: 'kv-platform-prod-001'
    location: 'eastus2'
    logAnalyticsWorkspaceResourceId: law.id
    tags: { environment: 'prod' }
  }
}
```

## Automation flow

1. **Detect** — `check-avm-version.sh` queries the public Bicep registry and
   compares the latest tag against `avm.version`.
2. **Diff** — `diff-avm-versions.sh` compiles old and new versions and emits
   added / removed / changed parameters and outputs.
3. **Dispatch** — A GitHub issue is opened and assigned to `@copilot` with the
   diff and explicit guardrails from
   [`.github/copilot-instructions.md`](./.github/copilot-instructions.md).
4. **Update** — The Copilot Coding Agent updates the wrapper, example
   parameter file, and README.
5. **Validate** — `tests/deploy.validation.sh` runs `bicep build`, `bicep lint`,
   and a sandbox what-if/deploy.
6. **Compliance** — SCF policy scan over the sandbox resource group.
7. **PR** — Pull request opened, Teams notified, human reviewer approves
   merge.

## Parameters

> Auto-generated by the automation. Edit `modules/keyvault.bicep` instead.

| Name | Type | Required | Default | Description |
| ---- | ---- | -------- | ------- | ----------- |
| `name` | string | yes | — | Key Vault name (3-24 chars) |
| `location` | string | yes | — | Azure region |
| `skuName` | string | no | `standard` | `standard` or `premium` |
| `enableRbacAuthorization` | bool | no | `true` | RBAC-only auth (locked) |
| `softDeleteRetentionInDays` | int | no | `90` | Soft delete retention (locked at 90) |
| `enablePurgeProtection` | bool | no | `true` | Purge protection (locked) |
| `networkAclsBypass` | string | no | `AzureServices` | Network ACL bypass |
| `networkAclsDefaultAction` | string | no | `Deny` | Default network action |
| `publicNetworkAccess` | string | no | `Disabled` | Public network access |
| `roleAssignments` | array | no | `[]` | RBAC role assignments |
| `privateEndpoints` | array | no | `[]` | Private endpoints |
| `diagnosticSettings` | array | no | `[]` | Diagnostic settings (overrides default) |
| `tags` | object | no | `{}` | Resource tags |
| `logAnalyticsWorkspaceResourceId` | string | no | `''` | Default diagnostic target |

## Outputs

| Name | Type | Description |
| ---- | ---- | ----------- |
| `resourceId` | string | Key Vault resource ID |
| `name` | string | Key Vault name |
| `uri` | string | Vault URI |
| `resourceGroupName` | string | Resource group name |
